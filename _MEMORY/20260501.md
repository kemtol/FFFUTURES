## Architecture Finding

Current data design risk:

- `DataBuffer` uses one final table `ohlcv_1m`.
- yfinance, Topstep, and backfill can effectively become mixed without source provenance.
- `INSERT OR IGNORE` means a bad candle can become sticky and not be corrected by a later cleaner source.

Recommended canonical architecture:

```text
raw_databento_1m   = historical/backfill authority
raw_yfinance_1m    = delayed append / continuity
raw_topstep_1m     = live / near-real-time gap-fill

canonical_ohlcv_1m = priority merge + source + quality_flag
```

Priority proposal:

1. Databento when available.
2. yfinance for matured bars if valid.
3. Topstep for newest bars or yfinance gaps.
4. Later yfinance/Databento can replace Topstep gap-fill if validated.
5. Obvious spikes should be rejected or flagged.
