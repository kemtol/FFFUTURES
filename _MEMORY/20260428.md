### тЬЕ Cycle 1: Pre-Breakout Profile Features тАФ KEPT

**Hypothesis**: The model sees a snapshot at breakout time (ATR, ADX, VWAP) but doesn't know *how the candle sequence led to the breakout*. A compression pattern (tightening range, low drift, inside bars) preceding the breakout signals a "coiled spring" тАФ more likely to follow through.

**Module**: [`pre_breakout_profile_features`](../pipeline/orb_ml/features/modules/generate_pre_breakout_profile_features.py) тАФ 5 features from 4├Ч15m pre-breakout candles:
- `pre_bo_compression_ratio` тАФ range of pre-bo candles / range of lookback candles
- `pre_bo_drift_atr` тАФ net price drift / ATR14
- `pre_bo_bullish_ratio` тАФ fraction of bullish pre-bo candles
- `pre_bo_inside_bar_flag` тАФ binary: was the last pre-bo candle an inside bar?
- `pre_bo_last_candle_range_ratio` тАФ last pre-bo candle range / avg pre-bo range

### тЬЕ Cycle 4: Macro Features тАФ KEPT

**Hypothesis**: ORB breakout outcomes are systematically affected by macro regime. When SPX is in a bull trend (above 200-day MA), DXY is weakening, US10Y yields are stable, and oil volatility is low, gold has a cleaner trend environment тЖТ breakouts follow through more reliably.

**Data**: First external data source. Fetched via [`yfinance`](../pipeline/fetch/fetch_macro_data.py) тАФ daily OHLC for SPY (SPX proxy), DX-Y.NYB (DXY), ^TNX (US10Y), CL=F (Oil). 4,258 rows, 2009-06-01 тЖТ 2026-04-28. Saved to [`macro_data.parquet`](../data/Level_1_Features/macro_data.parquet).

**Module**: [`macro_features`](../pipeline/orb_ml/features/modules/generate_macro_features.py) тАФ 4 features:
- `mac_spx_regime` тАФ SPX above (1) or below (0) 200-day SMA
- `mac_dxy_trend` тАФ DXY direction: +1 (bullish), 0 (neutral), -1 (bearish) vs 50-day SMA
- `mac_us10y_change` тАФ abs daily change in US10Y yield (regime instability signal)
- `mac_oil_volatility` тАФ abs daily return of Oil (energy shock / risk-off signal)

**Key design decisions**:
- Merge on `date` only (not full EVENT_KEY) тАФ macro regime affects all sessions in a day equally
- Forward-fill for weekends/holidays (last known macro state propagates)
- `date` dtype must be `datetime.date` (not str) for merge compatibility with other modules
