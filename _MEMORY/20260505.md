## Systemd Service

### tv_listener.service
```ini
[Service]
Type=simple
ExecStart=/usr/bin/python3 /home/kemal/futures/pipeline/live/run_tv_live.py
Restart=always
RestartSec=10
```

**Why systemd over nohup:** Terminal environment kills `nohup`/`disown` processes on disconnect. systemd `--user` services survive logouts and reboots.

**Commands:**
```bash
systemctl --user enable tv_listener     # auto-start on login
systemctl --user restart tv_listener    # restart after code changes
journalctl --user -u tv_listener -f     # live logs
```

### restart_daemons.sh
Post-reboot script launched via `@reboot cron`:
1. HTTP server (port 4173)
2. Feed daemon (TopstepX WS → SQLite)
3. TV listener (Super Structure auto-trade)
4. FVG listener (on deck, parked)

## Key Decisions

1. **TP_RISK_RATIO = 1.0 fixed for FVG** — sweep searches over SL, Session, Gap only
2. **No exchange stop orders for Super Structure** — SL is dynamic trailing SuperTrend, managed in code
3. **Combined buffer as single source** — MGC_1m.db + topstepx_buffer.db → combined_buffer.db (3.6M bars)
4. **systemd over nohup** — terminal environment unreliable, systemd survives everything
5. **Warmup 90→120 days** — DEMA(200) needs ~200 bars to converge, 120 trading days ≈ 34K bars
6. **Auto-initialize executor** — if `topstepx_token.json` exists, wire executor; else pure research mode
7. **FVG complement to ST** — ST magnitude strategy (low WR, high avg), FVG frequency (high WR, low avg)
8. **ORB v2.0 stays research-only** — no Telegram, no live execution, no subscribers

## What NOT to do

- Do not start FVG listener until tested on paper (parked in restart_daemons.sh)
- Do not run two executor instances simultaneously (would double-trade)
- Do not change `RestartSec=10` lower — rapid restart loops on crash
- Do not rename `topstepx_token.json` without re-initializing executor
- Do not modify `_heartbeat_state` keys without updating executor's `_format_heartbeat()`
- Do not create exchange stop/limit orders for Super Structure — SL is code-only
- Do not use `label="left"` in any resample — timestamps shift 5 min behind TradingView
