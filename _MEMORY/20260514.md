# 2026-05-14 — V8 Router Live Deployment

## Context

User asked whether the latest model's progress is reflected in the live
inference path. Investigation revealed major drift between validated artifacts
and what was actually running:

- Live (`pipeline/live/super_structure.py`) ran SMART_1 dual-ML
  (`regime_dispatcher.txt` + `conservative_brain.txt` + `aggressive_brain.txt`)
  with static thresholds `0.50/0.45` and no daily loss limit.
- Meta-v7 Refined Conservative was validated PASS Topstep solo (90d:
  PnL +$2,427 / max DD -$1,442 / 16.94% prob_of_ruin).
- SMART_1 combined dual-ML walk-forward 90d: PnL +$1,454 / DD -$2,175
  **FAIL MLL**, 42.6% prob_of_ruin.
- Aggressive ML brain was never validated at the setup that matched live —
  master sim used 8 features + threshold 0.55 + pullback events; live used
  10 features + threshold 0.45 + DEMA-cross events. Three-way inconsistency.

User intent (after walk-forward review): build a hybrid CONS ML
(Meta-v7 Refined) + AGGR mechanical (v1.12 pullback rules) that aligns sim
and live before any further trading.

## Design Decisions (D1-D5)

User-approved choices baked into the router:

- **D1**: Single-queue position (AGGR skipped when CONS in-position and vice
  versa). Simpler than dual position state; conservative.
- **D2**: New module `pipeline/live/pullback_detector.py` (not inline) for
  separation of concerns.
- **D3**: AGGR exits use v1.12 rules (SL = ST±1pt fixed, TP = 1R, 100-bar
  timeout). CONS keeps trend-flip + trailing SL. Mode tag determines path.
- **D4**: Single $700 combined daily cap (not $300 CONS + $400 AGGR split).
  Matches sim that produced the validated 90d numbers.
- **D5**: CT trading day for daily PnL (Topstep-aligned via 15h10m offset),
  not UTC calendar day. Sim was updated to match.

## Monitoring Alignment (also deployed 2026-05-14)

To ensure signals → Topstep → UI → parity all stay consistent with V8:

- **Heartbeat** (`pipeline/live/execute/super_structure_executor.py`) now
  renders a "🧠 V8 Router" block with daily PnL vs cap, current mode chip,
  last decision (PASS/SKIP + reason) every 5 min to Telegram.
- **Signal storage** (`_store_signal` in super_structure.py) now records
  `mode` ("CONS"/"AGGR"), `tp`, and `v8_router` boolean on every signal.
- **UI theoretical builder**
  (`pipeline/research/build_super_structure_trade_events.py`) now emits an
  AGGR pullback overlay alongside the existing DEMA-cross trades for the 5m
  output (102 events in current window). Each row tagged `mode`.
- **Parity check** (`pipeline/live/parity_super_structure.py`) now matches by
  `side + mode` so CONS signals only pair with CONS UI trades and AGGR with
  AGGR. JSON, Markdown, and Telegram comparison tables all include the mode
  column.

## Risk Posture

User explicitly accepted risk: this is a live account meant for testing.
Blown-up is acceptable; the primary goal is **alignment between signal →
Topstep → UI → parity**, not protecting principal. Daily PnL counter is
in-memory only (resets every restart) — by design.
